## Lenses

| Lens | Questions |
|------|-----------|
| Information retrieval | Are lexical, semantic, graph, and fact signals ranked correctly? |
| Graph theory | Are relationships meaningful, bounded, and explainable? |
| Systems | Will indexing and search scale without runaway IO or memory? |
| MCP behavior | Do tool schemas keep the model inside Loom? |
| Developer experience | Is setup boring, local, and obvious? |
| Benchmarking | Does the evidence measure useful symbols per token? |

## Method

1. Orient on actual code before theorizing.
2. Identify the current mechanism and failure mode.
3. Compare at least two plausible approaches when recommending a change.
4. Convert recommendations into concrete files, APIs, tests, and acceptance gates.
